Methodology

Old method (current calculator):

Effective TPM = RPM × (Input Tokens + Output Tokens)
GPU = ⌈ Effective TPM ÷ Old TPM-per-GPU benchmark ⌉
Cost = GPU × $36.50/day × Period multiplier

New method (weighted output):

Effective TPM = RPM × (Input Tokens + Multiplier × Output Tokens)
GPU = ⌈ Effective TPM ÷ New TPM-per-GPU benchmark ⌉
Cost = GPU × $36.50/day × Period multiplier

Why two benchmarks? The old benchmark measured raw token throughput per GPU. The new benchmark re-calibrates to "weighted tokens per GPU" using the same real telemetry. This ensures that for the platform's current average workload mix, old and new methods yield the same GPU count. But for new workloads with different output intensities, the weighted method correctly adjusts the GPU estimate.

Multiplier source: 5.1/5.2 chat & reasoning → 8× | 4.1 short-context → 4× (per Anna's follow-up). Validated against OpenAI public pricing ratios.

Benchmarks source: inferencedashboardlog.westus2/LLMAPI Kusto — 7-day rolling average as of March 2026. Cost: $36.50/GPU/day (A100e).
